template<class T>
Vertex< T > * Graph< T >::findVertex ( const T & in) const

Finds a vertex with a given content.

Parameters
inThe content to search for.
Returns
Pointer to the vertex, or nullptr if not found. \par Complexity:
O(V) linear search over the vertex set. This is the bottleneck in graph construction, where addEdge calls findVertex for every endpoint. A possible optimisation is to replace std::vector<Vertex<T>*> with std::unordered_map<T, Vertex<T>*>, reducing findVertex to O(1) and graph building to O(V + E).
